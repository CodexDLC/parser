# Development Plan

## Подход

Разрабатываем проект слоями. Сначала делаем работающий каркас и локальный запуск, потом добавляем внешние интеграции, потом автоматизацию.

## Этап 1. Каркас проекта

Цель: приложение запускается локально.

Сделать:

- структуру `src/aibot`;
- модульную структуру папок `src/aibot/db/`, `src/aibot/models/`, `src/aibot/api/routers/`, `src/aibot/api/schemas/`, `src/aibot/repositories/`, `src/aibot/services/`;
- `pyproject.toml`;
- `.python-version`;
- `src/aibot/main.py`;
- `src/aibot/config.py`;
- `/api/health`;
- README с базовым запуском.

Проверка:

- `uvicorn aibot.main:app --reload` запускается после установки проекта;
- `/docs` открывается;
- `/api/health` возвращает `{"status": "ok"}`.

## Этап 2. База данных и модели

Цель: сущности можно хранить и читать.

Сделать:

- подключение SQLAlchemy;
- подключение PostgreSQL;
- Docker Compose для PostgreSQL, если выбираем контейнерный локальный запуск;
- модели `Source`, `Keyword`, `NewsItem`, `Post`, `ErrorLog`;
- отдельный файл под каждую ORM-модель;
- отдельный repository под каждую основную сущность;
- Pydantic-схемы;
- отдельный файл схем под каждую группу сущностей;
- Alembic как единственный production-механизм изменения схемы;
- начальную versioned migration вместо production-вызова `create_all`.

Проверка:

- приложение стартует с пустой PostgreSQL-БД;
- `alembic upgrade head` создаёт таблицы;
- `alembic check` не видит расхождений между revision и ORM metadata;
- initial migration проходит downgrade и повторный upgrade;
- можно создать запись через код или API.
- интеграционный CRUD-тест проходит при поднятом `docker compose up -d postgres redis`.

## Этап 3. API управления

Цель: администратор может управлять источниками и фильтрами.

Сделать:

- CRUD `/api/sources/`;
- CRUD `/api/keywords/`;
- `GET /api/news/`;
- `GET /api/posts/`;
- `GET /api/logs/`.

Проверка:

- Swagger показывает все endpoints;
- source и keyword создаются, обновляются и читаются.

## Этап 4. Парсеры и дедупликация

Цель: сервис умеет получать новости.

Сделать:

- базовый интерфейс парсера;
- универсальный RSS/Atom parser с реальной HTTP-загрузкой;
- Telegram parser;
- сохранение `NewsItem`;
- `content_hash`;
- дедупликацию.

Проверка:

- ручной запуск парсинга создает новости;
- повторный запуск не создает дубли.

## Этап 5. Фильтрация

Цель: неподходящие новости не идут в генерацию.

Сделать:

- фильтр по разрешённым языкам;
- фильтр по ключевым словам;
- фильтр по enabled source;
- статусы `filtered_out` и `ready_for_generation`.

Проверка:

- новости на запрещённом или неопределённом языке отфильтровываются;
- новости без ключевых слов отфильтровываются;
- подходящие новости получают статус `ready_for_generation`.

## Этап 6. AI generation

Цель: можно получить готовый текст поста.

Сделать:

- AI client;
- prompt builder;
- ручной endpoint `/api/generate/`;
- генерацию `Post` из `NewsItem`;
- обработку ошибок.

Проверка:

- ручная генерация работает;
- пост сохраняется со статусом `generated`;
- ошибка AI не ломает приложение.

## Этап 7. Celery

Цель: тяжелые операции выполняются в фоне.

Сделать:

- Celery app;
- worker entrypoint;
- задачи parsing/generation/publishing;
- Celery Beat расписание на 30 минут;
- возврат `task_id` из ручных endpoints.

Проверка:

- worker запускается;
- задача парсинга выполняется;
- задача генерации выполняется.

## Этап 8. Telegram publishing

Цель: пост можно отправить в Telegram.

Сделать:

- publisher service;
- Telethon client;
- `POST /api/posts/{post_id}/publish`;
- dry-run режим;
- защиту от повторной публикации.

Проверка:

- dry-run не отправляет сообщение, но меняет статус предсказуемо;
- реальная публикация работает при заданных секретах;
- повторная публикация возвращает ошибку.

## Этап 9. Финализация сдачи

Цель: проект понятен преподавателю и проверяющему.

Сделать:

- полный README;
- `.env.example`;
- checklist с отметками;
- docker-compose, если успеваем;
- демонстрационный сценарий.

Проверка:

- новый человек может запустить проект по README;
- все обязательные пункты задания покрыты или явно отмечены как частично реализованные.
